# 변경 이력 (Changelog)

이 프로젝트의 주요 변경 사항을 기록합니다. 형식은 [Keep a Changelog](https://keepachangelog.com/ko/1.1.0/),
버전은 [유의적 버전(SemVer)](https://semver.org/lang/ko/)을 따릅니다.

배포: https://choicewise.kr

## [1.6.0] — 2026-07-18

### Added

- **가이드 기사 2편 추가** — 「기후동행카드 충전 종료, 이제 뭘 써야 하나」(충전 종료 7/31 대응, 전환 체크리스트),
  「청년미래적금 2차 접수 전 확인할 5가지」(2026년 12월 접수 대비, 자격·유형·은행·납입액·일정). 웹진 총 4편.
- **은행 비교 허브 선택 가이드** — 3%p/2%p 그룹 차이, 우대금리 3축(급여·카드·첫거래) 해설, '최고금리 낚시' 주의점과 FAQ.
- **K-패스 카드 비교 가이드** — 전월실적의 함정, 체크 vs 신용 선택 기준(상호 비교 링크), 카드별 한줄평과 FAQ.

### Changed

- **가이드 카드 통일** — 홈 하단과 가이드 목록의 기사 카드 디자인·접근성(제목 헤딩) 일치.
- **내비게이션 현재 위치 표시** — 상세 페이지에서도 해당 섹션 메뉴가 활성 표시되도록 통일.

## [1.5.0] — 2026-07-17

### Added

- **가이드 웹진** (`/guide`) — 청년 돈 문제를 계산기와 함께 짚는 장문 가이드 섹션 신설. 첫 기사 2편:
  「청도계 vs 청미적 — 갈아타기, 계산기로 3분 만에 판단하는 법」, 「모두의카드 추천 — K-패스 '모두의카드'는
  뭐고 누구에게 맞나」. 본문 안 링크로 계산기·비교 페이지로 바로 이동, 글 중간 CTA, 공식 출처(금융위·서민금융진흥원)
  링크와 바이라인 제공. 글로벌 내비 '가이드' 항목과 홈 하단 최신 가이드 카드 추가.
- **가이드 SEO** — 기사별 Article·BreadcrumbList·FAQPage 구조화 데이터, 프리렌더·사이트맵 등록.

### Changed

- **갈아타기 허용 시기 명시** — 특별중도해지 갈아타기가 2026-06 최초 가입기간 신청자 한정 일회성임을
  가이드 본문·FAQ에 명시(12월 재허용 여부는 공식 발표 전 미확정으로 안내).
- **애드센스 사이트 확인** — 소유권 확인 스니펫을 전 페이지에 추가(심사 기간 한시, 중복 로드 가드 포함).

### Fixed

- **다크모드 리스트 불릿** — 지원금 상세·가이드 본문의 불릿 점이 다크모드에서 보이지 않던 문제 수정.
- **본문 링크 안전성** — 가이드 마크다운 링크가 http(s)·사이트 내부 경로만 링크로 렌더되도록 제한.

## [1.4.0] — 2026-07-04

### Added

- **K-패스 교통카드 파인더** (`/transit`) — 지역·수단·대상·월교통비·카드종류·전월실적 6개 조건으로 2026 모두의카드
  실부담과 기후동행카드를 비교하고, 카드사 추가혜택 순위 TOP5를 추천하는 새 서비스. 3스텝 위저드(이용 조건 → 카드
  조건 → 결과) + 실물 카드 비율 타일의 가로 슬라이드, 공식 확인된 16개 카드는 `신청하러 가기` 링크 제공.
  순수 계산 엔진(`transitCardRec.ts`)은 정률/일반형/플러스형 3방식 최소 실부담 자동 적용, 전월실적 게이팅과
  **실적 구간별 상향 혜택 티어**까지 반영.
- **멀티서비스 플랫폼 전환** — react-router 기반 라우팅(`/` 청년적금, `/transit` K-패스), 공통 Shell(브랜드
  내비·푸터), 라우트별 SSG 프리렌더 + SEO 메타/JSON-LD 주입.
- **다크모드** — 상단 토글(시스템 설정 기본, 선호 저장), 새로고침 시 깜빡임 없는 초기 적용.
- **Pretendard 웹폰트** — 동적 서브셋 self-host.

### Changed

- **청년미래적금 1차 접수 마감 안내** — 신규 접수 마감(7/3)·다음 접수(2026-12) 배너와 SEO 문구 반영. 기가입자
  갈아타기·만기 계산은 계속 이용 가능.
- **K-패스 데이터 공식 확정** — 모두의카드 반값 기준금액(국토부 대광위), 기후동행 어르신 요금(서울시) 등 `[R]`
  미확정 값을 공식 출처로 확정하고 정본 카탈로그(`docs/transit-cards-research.md`) 신설. 비수도권 기준금액은
  미확정으로 남아 해당 지역 선택 시 추정치 안내를 표시.
- **접근성(WCAG AA)** — 브랜드 그린·보조 텍스트·뱃지·순위 메달 색을 4.5:1 이상으로 조정, 슬라이더 키보드
  포커스 링 추가.

### Fixed

- 손익분기 안내 폐기 — 모두의카드 캡 구조에서는 정률 기준 손익분기가 성립하지 않아 실부담 직접 비교로 일원화.
- 추천 배너 금액을 순위 기준(연회비 반영 순절감)과 일치시킴.

## [1.3.0] — 2026-06-30

### Added

- **결과 공유 링크** ([#19]) — 계산 상태(`inputs`+`leapStart`)를 URL `?s=` 파라미터로 직렬화/복원. 생년월일(PII)은
  싣지 않고, UTF-8 안전 base64 + URL-safe 인코딩, 손상·악의적 링크는 sanitize(비유한수 드롭·음수 0·경과/납입 0~만기
  클램프)로 방어. 공유 버튼과 링크 진입 시 자동 복원 포함.
- **단계형 입력 위저드** ([#19]) — 평면 `InputsPanel`을 스텝 위저드(`StepBadge`/`Slider`/`Segments`/`ToggleCard`)로
  교체. 모든 스텝을 마운트한 채 `hidden`으로 표시해 빌드 프리렌더의 크롤 가능성을 유지.

### Changed

- **결과 비교를 공정(3년 공통지평) 기준으로 개편** ([#19]) — 만기·납입한도가 다른 두 상품의 만기액 단순 비교 대신
  같은 월납입·3년 환산 비교를 메인 수치로, 각 상품 만기액은 참고 각주로 강등해 verdict와 일치시켰다.
- **결론 카드 보강** ([#19]) — 해지 순서 경고·미래적금 마감 D-day 노출, 공유 링크에서 생년월일 제거.
- 비교 카드 용어 표기 통일 — `실질효과율(광고)`·`정부기여금`·`미래적금`.

## [1.2.0] — 2026-06-24

### Changed

- **은행금리 모델 재설계** ([#6]) — 14개 기관 항목별 우대 %p·충족조건을 은행연합회 소비자포털 비교공시 정본으로
  재수집(전 기관 신뢰등급 `verified`). `bankRate`를 갈아타기 전용에 맞게 재구성 — **도약 연계가입·출시 우대를 전원
  자동가산**한 뒤 급여이체·카드·공통우대를 합산하고 기관 상한으로 캡. 기존 ‘주거래=기관상한’ 근사를 항목별 실제
  합산으로 정교화(예: NH농협 8.0% → 7.5%). 은행 데이터에 `salaryCond`/`switchCond`·`marketingReq`·`grade` 추가.

## [1.1.0] — 2026-06-24

### Added

- **추천 근거 비교표** ([#3]) — 유지 vs 전환 결론을 6개 판단 요인의 가중 점수로 분해해 표/카드로 노출.
- **추천 은행 + 선정 기준** ([#3]) — 전환 시 어떤 은행을 무슨 기준으로 골랐는지(적용금리 구성·우대 근거·순위) 명시.
- **은행 랭킹 상위 5개 + 전체 보기 토글** ([#4]).
- **광고 수익화 스캐폴드** ([#3], [#4]) — 애드센스/애드핏, 환경변수 기반 **기본 OFF**, `ads.txt`·개인정보처리방침 포함.
- **모바일 카드 레이아웃** ([#5]) — 좁은 화면에서 데이터 표를 카드로 전환.

### Changed

- **UI 전면 전환** ([#3]) — Tailwind CSS v4 + shadcn/ui(Radix)로 재구축(금융 디자인 토큰 유지).
- 빌드 툴체인 최신화 — Vite 8 / Vitest 4 / TypeScript 6 / ESLint 10, **Node 24 LTS**.

### Fixed

- 한글이 단어 중간에서 줄바꿈되던 문제 — `word-break: keep-all` ([#5]).

## [1.0.0] — 2026-06-24

### Added

- **최초 공개 배포** ([#1]) — 단일 HTML 프로토타입을 React 19 + TypeScript + Vite 정적 SPA로 재구성, Vercel 배포.
- 유지 vs 전환 순이익·실효수익률 비교, 거래현황 기반 최적 은행 추천, 소득구간·경과기간·유형별 시나리오 표.
- ‘적금 vs 투자’ break-even 비교, 전환 은행 수동 선택, 갈아타기 절차·순서 안내.
- 계산 엔진 순수 함수 분리 + Vitest 단위 테스트.

### Fixed

- number 입력에서 선행 0이 지워지지 않던 문제(`04000` → `4000`) ([#2]).
