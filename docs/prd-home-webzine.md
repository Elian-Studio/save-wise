# choicewise 홈 개편 + 웹진 PRD

기준일: **2026-07-15**  
상태: **Draft**  
범위: 첫 화면 홈 개편, 통합 청년 혜택 진단 진입, 웹진 영역 1차 추가

## 1. 배경

choicewise는 청년이 복잡한 금융·정책 선택을 빠르게 비교하도록 돕는 계산기형 서비스다. 현재 서비스 축은 다음 3개로 정리된다.

- **교통카드 추천**: K-패스·경기패스 등 교통비 절감 선택
- **청년적금 갈아타기**: 청년도약계좌 유지 vs 청년미래적금 전환 판단
- **청년지원금**: 받을 가능성이 있는 청년 정책·지원금 확인

현재 첫 화면은 교통카드 추천 중심으로 강하게 설계되어 있다. 다음 단계에서는 홈을 단일 서비스 랜딩이 아니라, 청년 사용자가 자신의 상황에 맞는 3개 서비스 중 어디로 가야 하는지 안내하는 **통합 진입면**으로 바꾼다.

## 2. 목표

### Product Goal

첫 방문자가 본인 상황을 간단히 입력하면 `교통카드 추천`, `청년적금 갈아타기`, `청년지원금` 중 우선 확인할 서비스를 추천받고, 필요한 경우 웹진 콘텐츠로 신뢰를 쌓은 뒤 각 서비스로 이동한다.

### User Goal

- 내가 지금 가장 먼저 확인해야 할 청년 혜택 영역을 30초 안에 안다.
- 복잡한 제도명을 몰라도 교통비, 적금, 지원금 중 어디서 돈을 아낄 수 있는지 이해한다.
- 최신 가이드성 콘텐츠를 통해 다시 방문할 이유를 얻는다.

### Business Goal

- 홈 이탈률을 줄이고 3개 핵심 서비스로의 진입을 분산한다.
- 검색 유입을 웹진 콘텐츠로 넓힌다.
- `choicewise = 청년 돈 아끼는 선택 도구`라는 포지션을 강화한다.

## 3. 핵심 결정사항

- 홈 메인 CTA는 **통합 청년 혜택 진단**으로 둔다.
- CTA 아래에는 3개 서비스 카드를 배치한다.
- 통합 진단 1차 버전은 상세 계산이 아니라 **3개 서비스 진입 우선순위 추천**만 제공한다.
- 진단 입력은 과도한 개인정보 없이 `나이`, `거주지`, `월 교통 이용`, `소득/취업 상태`, `저축 상품 보유 여부`로 제한한다.
- 웹진은 홈 하단에 **청년 돈 아끼는 가이드** 성격으로 두고 최신 글 3~4개를 노출한다.

## 4. 범위

### In Scope

- 홈 첫 화면 정보구조 개편
- 통합 청년 혜택 진단 CTA 및 진단 결과 진입 설계
- 3개 서비스 카드 노출
- 웹진 최신 글 3~4개 홈 노출
- 웹진 글 상세 페이지로 이어질 수 있는 기본 데이터 구조 설계
- 개인정보 최소화 및 면책 문구
- 모바일 우선 반응형 구조

### Out of Scope

- 회원가입, 로그인, 저장된 개인 프로필
- 주민등록번호, 계좌정보, 정확한 소득 증빙 등 민감정보 입력
- 3개 서비스 전체를 하나의 통합 계산기로 합치는 작업
- 청년지원금별 최종 신청 가능 여부 확정 판정
- 웹진 CMS, 관리자 글쓰기 화면
- 댓글, 좋아요, 북마크 등 커뮤니티 기능
- 광고 영역 재설계

## 5. 대상 사용자

### Primary

만 19~34세 청년. 교통비, 적금, 정책지원금을 확인하고 싶지만 어떤 제도부터 봐야 할지 모르는 사용자.

### Secondary

만 35~39세 청년성 제도 경계 사용자. 경기패스·인천 I-패스, 일부 청년지원금처럼 기준이 39세까지 확장되는 제도를 확인하려는 사용자.

### User Problems

- "K-패스, 경기패스, 청년미래적금, 지원금 이름은 들어봤는데 내게 맞는지 모르겠다."
- "정책 글은 많은데 실제로 내가 먼저 확인할 순서를 모르겠다."
- "여러 서비스가 있어도 첫 화면에서 어디를 눌러야 할지 모르겠다."

## 6. 홈 정보구조

### 6.1 Hero

목적: 첫 화면에서 choicewise의 정체성과 핵심 행동을 즉시 전달한다.

필수 요소:

- H1: 청년 돈 아끼는 선택, 30초 만에 먼저 볼 것부터
- Supporting copy: 교통비, 적금, 지원금을 한 번에 훑고 지금 확인할 서비스를 추천한다.
- Primary CTA: 통합 청년 혜택 진단 시작
- Secondary CTA: 3개 서비스 바로 보기
- 신뢰 고지: 공개 자료 기반 참고용, 최종 신청 전 공식 기관 확인 필요

### 6.2 통합 진단 진입

목적: 사용자가 서비스명을 몰라도 자기 상황 기반으로 진입하게 한다.

동작:

- CTA 클릭 시 홈 내 퀴즈 섹션 또는 `/guide` 성격의 진단 화면으로 이동한다.
- 5개 질문에 답하면 3개 서비스 우선순위를 보여준다.
- 결과는 "먼저 확인", "함께 보면 좋음", "나중에 확인" 3단계로 분류한다.
- 각 결과에는 한 줄 이유와 CTA를 붙인다.

예시:

- 먼저 확인: 교통카드 추천
- 이유: 수도권 거주 + 월 15회 이상 이용 가능성이 높아 교통비 환급 확인 우선
- CTA: 내 교통카드 찾기

### 6.3 서비스 카드

목적: 진단 없이 직접 들어오려는 사용자를 위한 명확한 선택지 제공.

카드 3개:

| 카드 | 핵심 문구 | CTA |
|---|---|---|
| 교통카드 추천 | K-패스·경기패스·후불카드 중 내 교통 패턴에 맞는 선택 | 교통카드 추천받기 |
| 청년적금 갈아타기 | 도약계좌를 유지할지, 미래적금으로 전환할지 비교 | 적금 갈아타기 계산 |
| 청년지원금 | 내가 받을 가능성이 있는 지원금과 정책을 먼저 확인 | 지원금 진단하기 |

카드 규칙:

- 각 카드는 1개 주요 CTA만 둔다.
- 수치·제도명은 검증된 데이터 문서와 코드의 단일 출처를 따른다.
- 신청 마감, 접수 예정, 종료 제도는 카드 안에서 과장하지 않는다.

### 6.4 웹진 영역

목적: 검색 유입과 재방문을 만들고, 서비스 사용 전후의 이해를 돕는다.

섹션명:

- 청년 돈 아끼는 가이드

노출 방식:

- 홈 하단에 최신 글 3~4개 노출
- 카드에는 `카테고리`, `제목`, `요약`, `업데이트일`, `읽는 시간`, `관련 서비스`를 표시
- 첫 버전은 정적 데이터 기반으로 시작한다.

초기 콘텐츠 후보:

| 카테고리 | 제목 후보 | 연결 서비스 |
|---|---|---|
| 교통비 | K-패스와 경기패스, 35세 이후엔 뭐가 달라질까 | 교통카드 추천 |
| 적금 | 청년미래적금 2차 접수 전 확인할 5가지 | 청년적금 갈아타기 |
| 지원금 | 청년지원금은 왜 신청기간이 제일 중요할까 | 청년지원금 |
| 공통 | 월급 들어오면 먼저 확인할 청년 혜택 체크리스트 | 통합 진단 |

콘텐츠 운영 원칙:

- 공식 출처 확인 전 금액·일정 단정 금지
- 종료된 제도는 "이미 종료"로 명확히 표시
- `2026`, `조건`, `신청기간`, `계산기`, `추천` 같은 검색 의도형 제목을 우선한다.

## 7. 통합 진단 1차 스펙

### 7.1 입력

| key | 질문 | 예시 옵션 |
|---|---|---|
| `age` | 만 나이가 어떻게 되나요? | 18세 이하, 19~24세, 25~29세, 30~34세, 35~39세, 40세 이상 |
| `region` | 주로 어디에 살거나 출발하나요? | 서울, 경기, 인천, 그 외 |
| `transitUsage` | 대중교통을 한 달에 얼마나 타나요? | 거의 안 탐, 15회 미만, 15~40회, 40회 이상 |
| `incomeJob` | 현재 소득/취업 상태는 어떤가요? | 학생/무직, 구직 중, 재직 중, 소득 있음·상세 모름 |
| `savingsStatus` | 청년도약계좌나 청년미래적금을 보유/관심 있나요? | 도약계좌 보유, 미래적금 관심, 둘 다 없음/모름 |

### 7.2 추천 로직 방향

정확한 자격 판정이 아니라 **서비스 우선순위 추천**이다.

- `transitUsage >= 15회`이면 교통카드 추천 우선도를 높인다.
- `region`이 경기/인천이면 교통카드 추천 이유에 지역형 패스를 포함한다.
- `savingsStatus`가 도약계좌 보유 또는 미래적금 관심이면 청년적금 갈아타기 우선도를 높인다.
- `incomeJob`이 구직 중, 학생/무직, 소득 있음·상세 모름이면 청년지원금 우선도를 높인다.
- `age`가 19~34세이면 3개 서비스 모두 기본 적합도를 높인다.
- `age`가 35~39세이면 경기/인천 교통카드와 일부 지원금 확인 가능성을 안내하되, 적금/지원금은 상세 화면에서 확인하도록 보수적으로 표현한다.
- `age`가 40세 이상이면 청년 특화 혜택은 제한될 수 있음을 표시하고, 교통카드 추천은 일반 이용자로 연결한다.

### 7.3 결과 화면

필수 요소:

- 추천 순위 1~3위
- 서비스별 한 줄 이유
- 서비스 CTA
- "정확한 금액·자격은 각 서비스에서 다시 계산" 고지
- 다시 진단하기

예시 문구:

- "먼저 볼 것: 교통카드 추천"
- "월 15회 이상 탈 가능성이 있어 K-패스 계열 환급부터 확인하는 게 좋아요."
- "다음: 청년지원금"
- "소득/취업 상태에 따라 확인할 지원금이 있을 수 있어요."

## 8. UX 원칙

- 첫 화면은 설명보다 행동을 우선한다.
- 카드형 나열은 3개 서비스 선택지에만 제한하고, 과도한 장식 섹션은 만들지 않는다.
- 모바일에서 CTA, 진단 질문, 서비스 카드가 한 화면 흐름으로 자연스럽게 이어져야 한다.
- 사용자가 제도명을 몰라도 이해할 수 있는 생활 언어를 쓴다.
- 단정적 금융·정책 자문처럼 보이지 않도록 "추천", "가능성", "확인" 표현을 쓴다.

## 9. SEO 요구사항

홈 타깃:

- 청년 혜택 추천
- 청년 돈 아끼는 법
- 청년 지원금 조회
- 청년 교통카드 추천
- 청년미래적금 갈아타기

웹진 타깃:

- 제도명 + 조건
- 제도명 + 신청기간
- 제도명 + 2026
- 청년 + 생활비 절약
- K-패스/경기패스/청년미래적금/청년지원금 관련 롱테일

메타 방향:

- 홈 title은 단일 서비스보다 통합 포지션을 반영한다.
- 웹진 글은 각 글의 검색 의도에 맞춘 독립 title/description을 가진다.

## 11. 성공 지표

### Launch Metrics

- 홈에서 통합 진단 CTA 클릭률
- 홈에서 3개 서비스 카드 클릭률
- 진단 완료율
- 진단 결과에서 각 서비스 CTA 클릭률
- 웹진 카드 클릭률

### Quality Metrics

- 모바일 첫 화면에서 메인 CTA가 스크롤 없이 보이는지
- 5문항 진단을 30초 안에 완료할 수 있는지
- 결과 문구가 자격 확정처럼 오해되지 않는지
- 종료·마감 제도 문구가 최신 데이터와 맞는지

## 12. 릴리스 기준

1차 릴리스 완료 조건:

- 홈 Hero에 통합 청년 혜택 진단 CTA가 노출된다.
- 3개 서비스 카드가 홈에서 명확히 보인다.
- 5문항 진단을 완료하면 3개 서비스 우선순위가 나온다.
- 각 추천 결과가 기존 서비스 경로로 연결된다.
- 웹진 최신 글 3~4개가 홈 하단에 노출된다.
- 모바일/데스크톱에서 텍스트 겹침, CTA 잘림, 카드 overflow가 없다.
- 공식 출처 재확인이 필요한 수치·일정은 단정하지 않는다.

## 13. Open Questions

- 통합 진단을 홈 내부 섹션으로 둘지, 별도 경로로 둘지 결정 필요.
- 웹진 상세 페이지를 1차 릴리스에 포함할지, 홈 노출 카드만 먼저 둘지 결정 필요.
- 홈 title을 기존 교통카드 SEO 중심에서 통합 청년 혜택 중심으로 바꿀지 결정 필요.
- 웹진 콘텐츠 발행 주기와 운영 책임자 결정 필요.
