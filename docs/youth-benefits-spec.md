## 1. 탑재 제도 (검증 완료)

### A. 자격 카드 (2026 신규 신청 가능) — 4개

#### A1. 청년내일저축계좌 `niljeo`  (category: 자산형성)
- 연령: 중위 50% 이하 = **만 15~39세**, 중위 50~100% = 만 19~34세. **단 2026년부터 차상위초과(50~100%) 신규모집 중단** → 신규 신청은 **중위 50% 이하 만 15~39세만**.
- 소득: 중위 50% 이하 & **월 근로·사업소득 10만원 이상**(정본 티어). (50~100% 티어는 신규중단이라 판정 제외)
- 지원: 본인 월 10만원 저축 시 정부 월 **30만원** 매칭, 3년 만기 정부적립 총 **1,080만원**.
- 신청: 복지로/읍면동. 출처: 복지로 자산형성포털 hope.welfareinfo.or.kr/bsns/bsnsIntrcnAcc.do (검증 high)
- 판정 입력: 나이, 가구 중위소득%(≤50), 본인 근로소득 유무(월10만↑).

#### A2. 국민취업지원제도 (청년 I유형 선발형) `chwiup`  (category: 취업)
- 연령: **만 15~34세**(병역이행 시 최대 **37세**).
- 소득: 가구 **기준중위소득 120% 이하**(I유형 선발형 청년특례). 재산: 가구 합산 **5억원 이하**.
- 상태: 미취업 구직자(실업급여·생계급여 수급자 제외).
- 지원: 구직촉진수당 월 **60만원 × 6개월** + 부양가족 1인당 월 10만(최대 40만) → 월 60~100만원.
- 신청: work24.go.kr. 출처: work24 systId=SI00000316, 정부24 (검증 high)
- 판정 입력: 나이(+병역), 가구 중위소득%(≤120), 재산(≤5억), 미취업 여부.

#### A3. 청년 주택드림 청약통장 `jutaek`  (category: 주거/자산)
- 연령: **만 19~34세**(병역기간 최대 6년 차감 후 34 이하 포함).
- 소득: 연소득 **5,000만원 이하** 신고소득자(근로·사업·기타).
- 상태: **무주택자**.
- 지원: 우대금리 청약통장(+연계 청년 주택드림 대출). **~2028-12-31까지 가입 가능**.
- 신청: 주택도시기금 수탁은행. 출처: nhuf.molit.go.kr/FP/FP07/FP0701/FP07010301.jsp (검증 high)
- 판정 입력: 나이(+병역), 연소득(≤5천만), 무주택 여부.

#### A4. 청년월세 특별지원(2차) `wolse`  (category: 주거)  ⚠️ 2026 접수 개시일 [R]
- 연령: 만 19~34세(병역 최대 6년 산입 예외), 부모와 **별도거주 무주택** 청년.
- 소득: **청년가구 중위 60% 이하 & 원가구 중위 100% 이하**. (30세↑·혼인·중위50%↑ 별도거주 등은 원가구 소득 미적용)
- 재산: 청년가구 **1억 2,200만원 이하** / 원가구 **4억 7,000만원 이하**.
- 지원: 월 최대 **20만원 × 12개월** 분할.
- 신청: 마이홈포털 myhome.go.kr. 출처: 국토부 사업매뉴얼·정부24 161300000099 (수치 검증 high)
- **`[R]` 2026년 신규 접수 개시 여부·일정 미확정** → 카드에 "2026 접수 일정은 마이홈포털에서 확인" 캡션 필수.
- 판정 입력: 나이, 별도거주+무주택, 청년가구 중위소득%(≤60), 재산.

### B. 내부링크 카드 (기존 서비스로 퍼널 — 수치 재진술 안 함)

## 2. 퀴즈 입력 스키마 (`QuizAnswers`)

자격 판정에 필요한 최소 입력. (passpick과 달리 소득/자산이 필요해 6~7문항)

| key | 질문 | 값 |

| `income` | 가구 세전 월소득 | 구간 버킷(중위소득% 산출) |

## 3. 엔진 (`src/lib/youthBenefitRec.ts`) — 순수 함수

`recommend(a: QuizAnswers): Result` — passpick `transitSchemeRec.ts` 미러.
- 각 제도에 대해 `eligible`(자격 규칙 전부 충족) / `likely`(대체로 충족, 경계·미확정 항목 있음 → "확인 필요") / `no` 판정 + `reasons[]` + `category` + CTA(internal|apply).
- 정렬: `eligible` → 지원규모(대략적 연 환산액) → category 순.
- 결과 그룹: **받을 수 있어요(eligible)** / **확인이 필요해요(likely·[R] 포함)** / **이미 종료된 제도(C)**.
- 월세의 `[R]` 접수여부, 미래적금 회차, 중위소득 `[R]` 테이블 미확정 시 해당 항목은 `likely`로 강등(절대 eligible로 단정 금지).
- 순수·SSR 결정적(입력 없을 때 기본값으로도 결정적 출력).

## 4. 검증
- `youthBenefitRec.test.ts`: 각 제도 자격 경계값(나이 컷 ±1, 중위소득% 상한 ±, 재산 상한, 무주택/미취업 플래그), 종료 제도는 항상 C그룹, 미확정 항목은 eligible로 안 뜸.
- 정직성 게이트: UI 노출 수치 = 본 문서 검증값과 1:1 대조. `[R]`(중위소득 테이블·월세 접수) 미해소 시 해당 카드 "확인 필요"로만 노출.
