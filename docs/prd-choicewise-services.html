제품 개요

choicewise(choicewise.kr)는 복잡한 정책·금융 상품의 "나에게 유리한 선택"을 계산해 주는 선택 도우미 계산기 플랫폼이다. 스펙 문서 → 순수 계산 엔진 → 단일 소스 데이터 → TDD 검증이라는 일관된 파이프라인으로 서비스를 늘려가며, 현재 2개 서비스를 운영·개발 중이다.

PRD 1 — 청년적금 갈아타기

문제 정의

2026년 청년미래적금 출시로 청년도약계좌 보유자는 "유지 vs 갈아타기"를 결정해야 한다. 두 상품은 만기(60개월 vs 36개월)·월 납입 한도·금리·정부기여금 구조가 달라 단순 만기수령액 비교는 오답을 낳는다. 특히 중도해지 시 기여금·비과세 처리(C2)에 따라 결론이 뒤집히는데, 이를 공정하게 계산해 주는 도구가 없다.

타겟 사용자

  • 갈아타기 모드: 청년도약계좌 기가입자 — 유지/전환 추천이 필요한 사용자.
  • 신규 모드: 청년미래적금 신규 가입 대상자 — 예상 수령액·최적 은행이 궁금한 사용자.

핵심 기능

  1. 두 경로 비교: 유지(도약 60개월 만기 원금+이자+기여금) vs 전환(지금 해지→해지환급금 + 미래적금 36개월 만기).
  2. 3축 공정 비교: ① 경로별 자체 만기 순수익·실효수익률 ② 공통 기간(36개월) 자산가치로 만기 불일치 보정 ③ 유동성·리스크 정성 평가(목돈최대 vs 유동성중시 goal 가중).
  3. 판정 출력: 추천 + 비교표 + 근거 3가지. 위저드 스텝 UI로 입력 수집.
  4. 신규 가입: 은행별 우대금리 반영 예상 수령액 랭킹.

이자 모델은 자유적립식 단리, 두 상품 모두 비과세(15.4% 면제).

범위

  • In: 도약↔미래 비교 판정, 신규 시뮬레이션, 은행별 랭킹, 결과 공유, SEO 리치 콘텐츠(FAQ·출처).
  • Out: 타 적금·투자상품 일반 비교, 회원/저장 기능, 실시간 금리 API(데이터 수동 갱신).

성공 기준

스펙(calc-spec.md)의 입출력 예시·경계값 테스트 전부 통과(현재 통과). 프리렌더 콘텐츠로 "청년미래적금 갈아타기" 계열 검색 유입 확보.

리스크

⚠️

C2(해지환급금 규정)가 최대 미확정 변수 — 중도해지 시 기여금·이자·비과세 보존/몰수 여부에 따라 추천이 뒤집힐 수 있다. [R] 값은 하드코딩하지 않고 파라미터로 유지, 확정 시 데이터만 교체.

PRD 2 — K-패스 교통카드 파인더

문제 정의

2026-01 K-패스가 "모두의카드"(정액제 무제한 환급)로 개편되며 환급 방식이 3가지(정률·일반형·플러스형)로 늘었고, 서울 거주자는 기후동행카드라는 대안도 있다. 최적 선택이 거주지·연령·이용 수단·월 교통비에 따라 달라지고, 카드사별 추가혜택(전월실적·연회비)까지 겹쳐 손으로 비교하기 어렵다.

타겟 사용자

대중교통 정기 이용자(월 교통비 2만~15만원). 특히 청년(만19~34)·저소득·경로 계층과 GTX·광역버스 이용 수도권 통근자.

핵심 기능

  1. 1차 — 제도 비교: 입력(거주지역·이용수단·대상유형·월 교통비)으로 K-패스 3방식 중 환급 최대 방식을 자동 적용한 실부담 vs 기후동행 정액 실부담을 비교, 승자 판정 + 손익분기 교통비 제시. 기후동행은 서울·비광역 수단만 가능 처리.
  2. 2차 — 카드 랭킹: K-패스 사용 시 카드사 추가혜택 기준 순비용 최소 카드 순위. 정률(캡·100원 내림)/정액 혜택, 전월실적 자격, 연회비 월환산 반영. 단종 카드 제외, 신용/체크 필터.
  3. 고지: "혜택 기준 2026-07 · 매월 변동 가능" 배지, 전월실적 충족 가정·공식 확인 필요 문구, 기후동행 서울 전용 각주.

범위

  • In: K-패스 vs 기후동행 비교, 손익분기, 카드사 랭킹(신용/체크), 기준일 고지.
  • Out: 알뜰교통카드 등 폐지 제도, 카드 발급 연동, 실시간 혜택 크롤링(데이터 수동 갱신).

성공 기준

transitCardRec.test.ts TDD 케이스 전부 통과: 3방식 최소 선택, 손익분기, flat/pct 계산, 단종 제외, 경계값(2만/15만, 지역별 가부) — 현재 통과. 로컬 검증 완료 후 배포하여 /transit 검색 유입 확보.

리스크

⚠️

[R] 미확정값: 비수도권 기준금액, 저소득 캡, 반값 종료(2026-10~) 후 정상값, 기후동행 senior 요금 — 모두 상수로 파라미터화, 확정 시 데이터만 교체. 코드 전체가 미커밋 워킹트리 상태 — feature 브랜치 커밋이 선행 과제.

공통 플랫폼 요구사항

  • 아키텍처: 계산 엔진은 순수 함수·입력 주도(src/lib/), 데이터는 단일 소스(src/data/)로 분리 — [R] 상수와 상품 테이블을 UI 수정 없이 교체 가능해야 한다.
  • SEO: 빌드 타임 SSG 프리렌더(scripts/prerender.mjs)로 라우트별 정적 HTML 생성, 라우트별 SEO 모듈(src/seo/) 관리.
  • 고지 의무: 모든 결과 화면에 "공개 자료 기반 참고용 추정, 투자·금융 자문 아님" 고지 유지.
  • 배포 규율: UI 변경은 로컬 dev 검증 완결 후 push(= Vercel 즉시 배포). 프로덕션에서 테스트·디버깅 금지.
