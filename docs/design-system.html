1. 디자인 원칙

코드·문구에서 관찰된 원칙만 명문화(발명 없음). 새 컴포넌트·색·카피 결정 시 판단 기준.

명료성 우선계산 결과가 주인공. navy 잉크로 숫자·결론을 또렷이, 근거는 <details>로 접어 노이즈 제거.
신뢰·정직“참고용 추정”, “보장되지 않습니다”, 기준일·출처 병기. 수익 단정·과장 금지 — 금융 자문이 아님.
한국어 우선 타이포word-break: keep-all로 어절 단위 줄바꿈. ‘NH농협’, ‘도약 월 3만원’ 같은 중간 깨짐 방지.
절제된 금융 팔레트토스풍 그린·그레이. 의미색(green=이득·blue=정보·amber=주의)만 강조에 쓰고 남발하지 않음.

2. 개요

  • 토큰 단일 소스는 src/index.css. tailwind.config.* / postcss.config.* 파일 없음.
  • Tailwind는 vite.config.ts의 @tailwindcss/vite 플러그인으로 로드 → 커스텀 토큰은 @theme inline 블록에만 존재.
  • spacing / breakpoint / (sans 외) fontFamily 확장 없음 → Tailwind v4 기본 스케일 사용.
  • 클래스 조합 유틸 cn() = twMerge(clsx(inputs)) (src/lib/utils.ts).

5. Components — 재사용 프리미티브

src/components/ui/* — 수동 작성 최소 shadcn 프리미티브. React 19 시그니처(React.ComponentProps), 아이콘 인라인 SVG(lucide 미사용). 모든 프리미티브가 className={cn('...base...', className)}로 오버라이드 허용. cva variant는 Button·Badge 2곳뿐.

Button (ui/button.tsx)

size: default h-9 px-4 py-2 · sm h-8 px-3 text-xs · lg h-10 px-6. 상태: hover(default bg-primary/90), focus-visible:ring-2 ring-ring, disabled:opacity-50 pointer-events-none. asChild(Radix Slot)로 링크를 버튼처럼.

When to use주 행동 1개엔 default(primary). 보조 행동은 outline/secondary, 최소 강조는 ghost.
Don't한 화면에 default 버튼 여러 개로 위계 흐리기. 링크 대신 버튼 남용. 텍스트만 다른 커스텀 색 새로 만들기.

Badge (ui/badge.tsx)

default secondary outline green blue amber muted

base rounded-md px-2 py-0.5 text-xs font-bold. green/blue/amber는 금융 시맨틱 soft 배경.

When to use의미대로: green=이득/절약, blue=정보/보조, amber=주의, muted=중립 라벨. 상태·분류를 짧게 표시.
Don't의미와 다른 색 쓰기(주의에 green). 긴 문장·클릭 액션을 뱃지에 담기. green/blue를 본문급 작은 글자로 — 대비 미달(§7).

그 외 프리미티브 & 위저드 컨트롤

cva 없음 (단일 cn() 문자열): Radix 래핑 select/tabs/checkbox/label, 순수 HTML 래퍼 card/input/table(overflow-x-auto).

위저드 전용 wizard/controls.tsx (StepBadge/Slider/Segments/ToggleCard)는 cva 대신 삼항(active ? … : …)으로 variant 처리 — ui/ 패턴과 별개.

7. 반응형 · 접근성 · 모션

  • breakpoint: sm: / lg:만. 커스텀 없음(v4 기본). 모바일 1단 → 데스크톱 2단, 데스크톱 lg:sticky lg:top-4.
  • 접근성/터치: 44px 터치 타깃(min-h-11), 슬라이더 thumb 24px(input.range), 표 overflow-x-auto, 포커스 링 focus-visible:ring-ring, 시맨틱 <nav>/<header>/<footer> + aria-current.
  • 모션: 라이브러리 없음, tw-animate-css만. 위저드 스텝 animate-in fade-in slide-in-from-bottom-3 duration-300 motion-reduce:animate-none, Select data-[state=open]:animate-in.

색 대비 (WCAG 2.2 AA) — 실측

sRGB relative luminance 기준 실측. 본문 텍스트 AA=4.5:1, 큰 글자(≥18.66px bold/≥24px)=3:1. text-xs(12px) bold는 “큰 글자” 아님 → 4.5 필요.

전경 / 배경대비판정
foreground #191f28 / background #edf1f514.59AA ✔
accent-fg #0b6b4f / accent #e7f7f15.87AA ✔
fin-amber #b45309 / amber-soft #fdf2e2 (Badge amber)4.54AA ✔
fin-blue #3182f6 / blue-soft #eaf2fe (Badge blue)3.29큰 글자만, 본문 ✘
muted-foreground #8b95a1 / background #edf1f52.68AA ✘
fin-green #00a860 / green-soft #e7f7f1 (Badge green)2.80AA ✘
primary-fg #ffffff / primary #12b886 (default 버튼)2.55AA ✘

브랜드 색 유지가 전제라 코드는 미변경. 보완책: 미달 색은 큰 글자·아이콘·테두리 병행으로 정보를 색에만 싣지 않기, primary 버튼은 대비 확보를 원하면 잉크(navy) 텍스트 또는 더 진한 그린 검토. 결정 전까지 알려진 제약으로 관리.

8. Content — 보이스 & 톤

금융 계산 서비스 카피 규칙. Disclaimer.tsx·as-of pill 관행에서 도출.

Do“참고용 추정”·가정 명시, 기준일(DATA_AS_OF)·출처(은행연합회 정본) 병기, 숫자·단위·%p 정확히, 가입 전 재확인 안내.

  • 시맨틱 버전·리뷰 SLA·전용 툴(zeroheight 등)은 현 규모에 불필요. 기여자가 늘면 그때 도입.

10. Known gaps

문서화 중 발견한 코드-문서 불일치. 기록만 — 코드 미수정.

Pretendard 미로드 — --font-sans 1순위인데 index.html에 <link>·@font-face·웹폰트 파일 전무. 대부분 환경에서 시스템 폰트 폴백. 의도면 스택에서 제거, 아니면 CDN/self-host 추가.
다크모드 미구현 — @custom-variant dark (&:is(.dark *)) 선언만 있고 .dark 팔레트 오버라이드 블록 없음 → dark: 유틸 사실상 무효.
미사용 레거시 컴포넌트 — VerdictCard(테스트만)/DecisionTable/ComparePanels/BankPick/MiraeSummary/StepsGuide. 라우팅 도달 불가. 신규 작업 참고 대상 제외.
접근성: 주요 색 대비 AA 미달 — Badge green(2.8)·blue(3.29), muted-foreground on bg(2.68), default 버튼 흰 글자 on primary(2.55)가 WCAG AA(4.5:1) 미달(§7 실측표). 브랜드 색 유지 시 색 외 단서(굵기·아이콘·테두리) 병행 권장.
